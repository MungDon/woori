* 자바스크립트 코딩 규칙 - 스타일 가이드, 코딩 컨벤션(관습이나 규칙), 코딩 스타일, 표준 스타일 - 웹 문서에 동적인 효과를 주기 위해 출발한 언어 - 다른 프로그래밍 언어에 비해 타입(type)이 유연해서 작성자가 주의를 기울이지 않으면 오류가 발생함 - 스타일 가이드에 맞게 작성하면 코드의 오류도 줄이고 일관성이 생겨 가독성이 좋아진다. 웹 사이트나 애플리케이션의 유지보수 시간, 비용 절감 효과. * 소스 작성 규칙 1. 들여쓰기 2. 세미콜론으로 문장을 구분한다. 3. 공백을 넣어서 읽기 쉽게 작성한다. 4. 코드 설명 주석(comment) 작성. // 한줄주석 /* 여러줄 주석 */ 5. 식별자 : identifier : 개발자가 변수, 함수, 속성 등을 구별하려고 붙인 특정 단어 - 첫글자 반드시 소문자이거나 _, $ 사용가능 - 두단어의 연결 - , _, 뒷단어 첫글자 대문자 6. 예약어는 식별자로 사용 금지 : keyword - 자바 스크립트에서 미리 정해 놓은 단어 예) var 변수 선언시 사용 예약어 - > 식별자로 사용 불가능
